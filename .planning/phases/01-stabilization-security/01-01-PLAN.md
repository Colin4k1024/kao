# Phase 01 Plan 01: Critical Security Vulnerabilities Fix

## Objective

Fix critical security vulnerabilities by removing hardcoded credentials, fixing JWT secrets, and replacing password bypass with proper bcrypt verification.

## Purpose

This is the foundation phase for security. Without these fixes, the system is vulnerable to unauthorized access. Phase 1 success requires zero hardcoded credentials or secrets.

## Output

Secure authentication implementation with bcrypt + JWT, proper environment variable configuration, and consolidated auth flow.

### Codebase References

**Current Issues (from CONCERNS.md):**

1. **app.rs (line 45-47):** Hardcoded admin credentials bypassing JWT
   ```rust
   // ISSUE: Hardcoded admin check
   if username == "admin" && password == "admin123" {
       return Ok(admin_token);
   }
   ```

2. **extractor.rs (line 43):** Hardcoded JWT secret
   ```rust
   // ISSUE: Hardcoded secret
   let secret = "change-me-in-development";
   ```

3. **handlers.rs (line 55):** Simple string comparison instead of bcrypt
   ```rust
   // ISSUE: Password bypass
   if req.password == "admin123" {
       // login fails, but if password matches hardcoded, succeeds
   }
   ```
