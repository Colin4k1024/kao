# Phase 06 Plan 01: Job Scheduler Module

**Phase:** 06-job-scheduler

## Objective

Implement the missing job scheduling backend module to support the existing frontend job management UI.

**Deliverables:**
1. Job CRUD API (list, get, create, update, delete)
2. Job scheduling control (schedule, unschedule, run-once)
3. Job log API (list, get, clear)

---

## Background

The frontend has a complete job management UI at `frontend/src/pages/job/` that expects these endpoints:
- GET `/api/system/jobs` - list jobs
- GET `/api/system/jobs/:id` - get job
- POST `/api/system/jobs` - create job
- PUT `/api/system/jobs/:id` - update job
- DELETE `/api/system/jobs/:id` - delete job
- PUT `/api/system/jobs/:id/schedule` - schedule job
- PUT `/api/system/jobs/:id/unschedule` - unschedule job
- POST `/api/system/jobs/:id/run` - run once
- GET `/api/system/jobs/logs` - list job logs
- GET `/api/system/jobs/logs/:id` - get log
- DELETE `/api/system/jobs/logs/clear` - clear logs

The backend `job` module does not exist - this is the primary gap.
