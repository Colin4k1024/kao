# Database Reference

The runtime source of truth is the SQL migration set under `backend/migrations/`.

`database/schema.prisma` is kept as a reference model for the domain shape, naming, and relationship intent. It should be updated alongside migrations, but the application should not depend on Prisma Client or generated Prisma artifacts for runtime persistence in this framework.

The SQL schema intentionally avoids requiring a database UUID extension by default. IDs are expected to be generated by the application layer or supplied explicitly in seed data.

Current phase-one scope:

- users
- departments
- roles
- menus
- user-role links
- role-menu links
- role-department links

Future phase work can add posts, dictionaries, parameters, notices, jobs, and generator metadata if needed.
